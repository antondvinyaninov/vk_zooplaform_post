# UI Rules (Admin + VK Mini App)

Этот файл фиксирует обязательные UI-правила проекта.

## Обязательное правило для списков

1. Любое поле выбора из списка (роль, статус, фильтр, группа и т.д.) должно использовать компонент **VKUI CustomSelect**.
2. Нативный HTML `<select>` в продуктовых экранах не использовать.
3. Для новых экранов и рефакторинга существующих экранов:
   - сначала закладываем `CustomSelect` как стандартный путь;
   - только временно (в прототипе) допустим fallback, который должен быть заменен на `CustomSelect`.
4. Выпадающее меню `CustomSelect` всегда должно открываться **поверх других блоков** и не обрезаться контейнерами.
5. Запрещено клиппирование dropdown через `overflow: hidden` у родительских контейнеров формы/карточки.
6. Для dropdown используем отдельный слой (высокий `z-index`; при необходимости portal/overlay-подход).

Ссылка на компонент: https://vkui.io/components/custom-select/

## Scope

Правило действует для:
- `vk_app/` (React + VKUI)
- `frontend/` (админка)

## Почему

- единое поведение во всех формах;
- консистентный UX в стиле VKUI;
- проще поддержка дизайна и QA.
