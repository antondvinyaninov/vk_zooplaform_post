# Концепция развития: ЗооПлатформа как "Приложение-Хамелеон" и Агрегатор

## Основная идея (SuperApp)

Вместо создания множества разрозненных VK Mini Apps для разных типов зоо-организаций (приюты, ветклиники, группы объявлений), мы создаем **одно универсальное приложение**. 

Приложение способно динамически менять свой интерфейс и функционал в зависимости от того, **откуда** (в каком контексте) его открыл пользователь.

---

## Архитектура 2-в-1: Локальный и Глобальный режимы

Ключевая особенность архитектуры VK Mini Apps позволяет нашему приложению работать в двух принципиально разных режимах, используя одну и ту же базу данных.

### 1. Локальный режим (White-label CRM)
**Условие запуска:** Приложение открыто из меню конкретного сообщества ВКонтакте (передан параметр `vk_group_id`).

**Как это работает:**
* Наш бэкенд определяет тип сообщества (Приют, Ветклиника, Группа помощи).
* Приложение "перекрашивается" под нужды этой группы и становится её личным сайтом / CRM-системой.
* **Для Приюта:** Открывается локальный каталог животных, которые находятся именно в этом приюте. Формы пожертвований и заявки на усыновление отправляются администраторам этого приюта.
* **Для Ветклиники:** Открывается перечень услуг этой клиники и онлайн-запись к их врачам.
* **Для Группы объявлений:** Открывается привычный интерфейс "Предложить пост" (потеряшки/находки).

**Выгода:** Зоо-организации получают мощный бесплатный инструмент (CRM) внутри своей группы без необходимости нанимать программистов.

### 2. Глобальный режим (Маркетплейс / Агрегатор)
**Условие запуска:** Приложение открыто напрямую из каталога сервисов ВКонтакте или по прямой ссылке (параметр `vk_group_id` отсутствует).

**Как это работает:**
* Приложение понимает, что оно работает вне группы, и превращается в **Глобальную ЗооПлатформу**.
* **Единый каталог животных (Тиндер для питомцев):** Пользователь видит животных, ищущих дом, собранных из **всех** подключенных приютов города.
* **Карта ветклиник:** Единый портал всех партнерских клиник с возможностью записаться в любую из них.
* **Глобальная лента потеряшек:** Все объявления о пропавших животных, агрегированные со всех городских групп.

**Выгода:** Это Win-Win. Приюты и клиники, просто используя нашу локальную CRM для своих нужд, автоматически выгружают свои данные в глобальный агрегатор, откуда мы можем наливать им огромный бесплатный трафик. А ЗооПлатформа получает стремительно растущую единую базу данных животных по всей стране.

## Техническая реализация (Бэкенд)

1. **Единое API и База данных:** Вся логика обрабатывается существующим бэкендом на Go.
2. **Маршрутизация на Фронтенде:** React-приложение (VKUI) имеет корневой переключатель (Router), который оценивает параметр `vk_group_id` и флаг `group_type` из базы данных, загружая соответствующий `Epic` (модуль интерфейса).
3. **Онбординг администраторов:** При первой установке приложения в группу, администратор проходит быстрый опрос (Выбор типа: Приют/Клиника/Волонтеры), после чего приложение навсегда фиксирует свой локальный интерфейс для этой группы.
