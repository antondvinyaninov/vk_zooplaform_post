# Архитектура проекта

## Общая схема

```
┌─────────────────────────────────────────────────────────────┐
│                      VK Post Platform                        │
└─────────────────────────────────────────────────────────────┘

┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  VK Mini App │    │   Админка    │    │ Основной сайт│
│  (vk_app/)   │    │ (frontend/)  │    │   (site/)    │
└──────┬───────┘    └──────┬───────┘    └──────┬───────┘
       │                   │                   │
       └───────────────────┼───────────────────┘
                           │
                    ┌──────▼──────┐
                    │  Go Backend │
                    │ (backend/)  │
                    └──────┬──────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   ┌────▼────┐      ┌──────▼──────┐    ┌─────▼─────┐
   │ SQLite  │      │   VK API    │    │  External │
   │   DB    │      │   Client    │    │    APIs   │
   └─────────┘      └─────────────┘    └───────────┘
```

## Компоненты системы

### 1. Frontend слой

#### VK Mini App (`vk_app/`)
- **Технологии**: React 18, TypeScript, VKUI 8, VK Bridge
- **Назначение**: Пользовательский интерфейс для создания постов
- **Функционал**:
  - Авторизация через VK Bridge
  - Создание и редактирование постов
  - Загрузка медиа (фото, видео)
  - Предпросмотр постов
  - Публикация в группы

#### Админка (`frontend/`)
- **Технологии**: HTML, JavaScript, CSS
- **Назначение**: Управление системой
- **Функционал**:
  - Подключение групп ВКонтакте
  - Управление токенами доступа
  - Настройки постинга
  - Статистика публикаций
  - Управление пользователями

### 2. Backend слой (`backend/`)

#### Структура модулей

```
backend/
├── admin/          # API для админки
├── vk-app/         # API для VK Mini App
├── site/           # API для основного сайта
├── api/
│   ├── external/   # Внешние API (VK, Telegram)
│   └── internal/   # Внутренние API (БД)
├── vk/             # VK API клиент
├── database/       # Работа с SQLite
├── models/         # Модели данных
├── middleware/     # Middleware
├── config/         # Конфигурация
├── services/       # Бизнес-логика
└── utils/          # Утилиты
```

#### Разделение ответственности

**admin/** - Админка API
- Управление группами
- Управление пользователями
- Статистика
- Настройки системы

**vk-app/** - VK Mini App API
- Создание постов
- Публикация в группы
- Получение списка групп
- Работа с медиа

**site/** - Основной сайт API
- Публичные endpoints
- Контактная форма
- Подписка на новости

**api/external/** - Внешние API
- VK API клиент
- Telegram API (будущее)
- Другие интеграции

**api/internal/** - Внутренние API
- CRUD операции с БД
- Бизнес-логика
- Валидация данных

### 3. Слой данных

#### SQLite Database
- **Таблицы**: groups, posts, users
- **Индексы**: Оптимизированы для частых запросов
- **Миграции**: Автоматические при запуске

#### Модели данных
- `Group` - Группы ВКонтакте
- `Post` - Посты (draft, scheduled, published)
- `User` - Пользователи системы

### 4. Интеграции

#### VK API
- Публикация постов
- Загрузка медиа
- Получение информации о группах
- Получение информации о пользователях

#### VK Bridge
- Авторизация в Mini App
- Получение токенов
- Работа с VK UI

## Потоки данных

### Публикация поста

```
1. Пользователь создает пост в VK Mini App
   ↓
2. VK Mini App → POST /api/app/posts/create
   ↓
3. Backend сохраняет пост в БД (status: draft)
   ↓
4. Backend загружает медиа через VK API
   ↓
5. Backend публикует пост через VK API
   ↓
6. Backend обновляет статус (status: published)
   ↓
7. Ответ возвращается в VK Mini App
```

### Подключение группы

```
1. Админ авторизуется в админке
   ↓
2. Админка → POST /api/vk/groups (с токеном)
   ↓
3. Backend получает список групп через VK API
   ↓
4. Админ выбирает группы для подключения
   ↓
5. Backend сохраняет группы в БД
   ↓
6. Группы доступны для публикации в VK Mini App
```

## Безопасность

### Авторизация
- VK Bridge для Mini App
- OAuth для админки
- Токены хранятся в БД

### Валидация
- Проверка токенов на каждый запрос
- Валидация данных на backend
- Санитизация входных данных

### CORS
- Настроен для всех endpoints
- Разрешены только нужные origins

## Масштабирование

### Текущая архитектура
- Один Go процесс
- SQLite база данных
- Подходит для малых и средних нагрузок

### Будущее масштабирование
- Миграция на PostgreSQL
- Горизонтальное масштабирование backend
- Redis для кэширования
- Очереди для отложенной публикации

## Мониторинг

### Логирование
- Middleware логирует все запросы
- Ошибки логируются с stack trace
- Логи доступны через stdout

### Метрики (будущее)
- Количество публикаций
- Время ответа API
- Ошибки VK API
- Использование БД

## Деплой

### Development
```bash
# Backend
cd backend && go run main.go

# VK Mini App
cd vk_app && npm start
```

### Production
```bash
# Docker
docker build -t vk-post-platform .
docker run -p 80:80 vk-post-platform
```
