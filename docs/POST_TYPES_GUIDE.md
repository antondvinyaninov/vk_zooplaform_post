# Руководство по типам объявлений

## Обзор

Система типов объявлений позволяет настраивать категории и пользовательские поля для объявлений в каждой группе ВКонтакте.

## Типы полей

### 1. **Строка** (`string`)
Обычное текстовое поле для ввода текста.

**Примеры использования:**
- Кличка животного
- Порода
- Окрас
- Адрес

**В форме VK Mini App:**
```tsx
<input type="text" placeholder="Введите кличку" />
```

**В посте VK:**
```
Кличка: Барсик
Порода: Дворняга
```

---

### 2. **Ссылка** (`link`)
Поле для ввода URL с валидацией.

**Примеры использования:**
- Сайт приюта
- Ссылка на соцсети
- Дополнительная информация

**В форме VK Mini App:**
```tsx
<input type="url" placeholder="https://example.com" />
```

**В посте VK:**
```
Сайт приюта: https://shelter.ru
```
*VK автоматически делает ссылки кликабельными!*

---

### 3. **Число** (`number`)
Числовое поле с валидацией.

**Примеры использования:**
- Возраст (лет)
- Вес (кг)
- Цена (₽)

**В форме VK Mini App:**
```tsx
<input type="number" placeholder="Введите возраст" />
```

**В посте VK:**
```
Возраст: 3 года
Вес: 15 кг
```

---

### 4. **Телефон** (`phone`)
Поле для ввода телефона с форматированием.

**Примеры использования:**
- Контактный телефон
- Телефон приюта

**В форме VK Mini App:**
```tsx
<input type="tel" placeholder="+7 (999) 123-45-67" />
```

**В посте VK:**
```
Контакт: +7 (999) 123-45-67
```
*VK автоматически делает телефоны кликабельными!*

---

### 5. **Выбор из списка** (`select`) ⭐ НОВОЕ
Выпадающий список с заранее заданными вариантами.

**Примеры использования:**
- Пол: Мальчик / Девочка / Неизвестно
- Размер: Маленький / Средний / Большой
- Возраст: Щенок / Молодой / Взрослый / Пожилой

**Настройка в админке:**
```json
{
  "id": "gender",
  "label": "Пол",
  "type": "select",
  "required": true,
  "options": ["Мальчик", "Девочка", "Неизвестно"]
}
```

**В форме VK Mini App:**
```tsx
<select>
  <option>Мальчик</option>
  <option>Девочка</option>
  <option>Неизвестно</option>
</select>
```

**В посте VK:**
```
Пол: Мальчик
```

---

### 6. **Да/Нет** (`boolean`) ⭐ НОВОЕ
Чекбокс для бинарного выбора.

**Примеры использования:**
- Стерилизован
- Привит
- Чипирован
- Приучен к лотку

**Настройка в админке:**
```json
{
  "id": "sterilized",
  "label": "Стерилизован",
  "type": "boolean",
  "required": false
}
```

**В форме VK Mini App:**
```tsx
<input type="checkbox" /> Стерилизован
```

**В посте VK (если отмечено):**
```
✓ Стерилизован
```

**В посте VK (если не отмечено):**
```
(поле не отображается)
```

## Как настроить в VK Mini App

1. Откройте VK Mini App
2. Перейдите в **Профиль** → **Настройки сообщества**
3. Включите переключатель **Использовать категории и анкеты**
4. Нажмите **+ Добавить категорию**
5. Заполните:
   - Название категории (например: "Собаки")
   - Цвет (нажмите на цветной кружок)
6. Нажмите на иконку **списка** рядом с категорией для добавления полей
7. Для каждого поля укажите:
   - Название поля
   - Переменная (латиницей, например: phone_1)
   - Тип поля
   - Обязательное или нет
   - Для типа "Выбор из списка" - добавьте варианты
8. Нажмите **Сохранить настройки**

## Технические детали

### Структура данных в БД

**Таблица `groups`:**
```sql
enable_post_types BOOLEAN DEFAULT FALSE
post_types JSONB
```

**Таблица `post_publications`:**
```sql
post_type_id TEXT
custom_fields JSONB
```

### API Endpoints

**Получить настройки группы:**
```
GET /api/admin/groups/:id
```

**Обновить настройки группы:**
```
PATCH /api/admin/groups/:id
Body: {
  "enable_post_types": true,
  "post_types": [...]
}
```

**Создать пост с кастомными полями:**
```
POST /api/app/posts/create
Body: {
  "message": "...",
  "post_type_id": "dogs",
  "custom_fields": [
    {"id": "name", "value": "Барсик"},
    {"id": "gender", "value": "Мальчик"},
    {"id": "sterilized", "value": "true"}
  ]
}
```

## Лучшие практики

### ✅ Рекомендуется:
- Использовать **select** для полей с ограниченным набором вариантов
- Использовать **boolean** для простых да/нет вопросов
- Делать обязательными только критичные поля
- Давать понятные названия полям
- Группировать похожие типы объявлений

### ❌ Не рекомендуется:
- Создавать слишком много полей (оптимально 5-8)
- Использовать **string** там, где подходит **select**
- Делать все поля обязательными
- Использовать технические названия полей

## FAQ

**Q: Можно ли изменить тип поля после создания?**  
A: Да, но это может привести к потере данных в существующих постах.

**Q: Сколько вариантов можно добавить в select?**  
A: Рекомендуется не более 10 вариантов для удобства пользователей.

**Q: Как отображаются boolean поля в посте?**  
A: Если отмечено - отображается "✓ Название поля", если нет - не отображается.

**Q: Можно ли использовать emoji в названиях?**  
A: Да, emoji поддерживаются во всех текстовых полях.

**Q: Как удалить тип объявления?**  
A: Нажмите на иконку корзины рядом с типом в настройках группы.
