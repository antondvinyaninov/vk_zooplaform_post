# VK Bridge: Получение токенов доступа (Auth Token & Community Token)

Помимо классического OAuth (описанного в `VK_AUTH.md`), ВКонтакте предоставляет нативные методы для получения токенов пользователя и сообщества прямо внутри VK Mini Apps через `VK Bridge`.

## 1. Получение пользовательского токена (`VKWebAppGetAuthToken`)

Позволяет запросить у пользователя права доступа (scope) и получить `access_token` для вызова методов VK API от лица пользователя.

**Сценарии использования:**
- Разрешить пользователю публиковать посты на своей личной стене прямо из Mini App.
- Авторизовать администратора внутри Mini App для выполнения действий (модерация, управление товарами, публикация), не заставляя его переходить в отдельную веб-админку.

**Пример вызова:**
```typescript
import bridge from '@vkontakte/vk-bridge';

bridge.send('VKWebAppGetAuthToken', {
  app_id: 54555042, // ID нашего приложения
  scope: 'wall,photos,video,groups'
})
.then((data) => {
  if (data.access_token) {
    console.log('Пользовательский токен получен:', data.access_token);
    // Можно отправить токен на наш бэкенд или делать вызовы к VK API прямо с клиента
  }
})
.catch((error) => {
  console.log('Ошибка получения токена или отказ пользователя:', error);
});
```

## 2. Получение токена сообщества (`VKWebAppGetCommunityToken`)

Позволяет запросить у администратора сообщества доступ (токен сообщества) для вашего приложения. Этот токен используется для вызова методов API от имени группы (например, для публикации постов, настройки виджетов или отправки сообщений от имени группы).

**Сценарии использования:**
- **Onboarding (самое важное!):** Когда администратор впервые открывает приложение в своей группе, мы можем прямо там, с помощью одной кнопки, запросить `Community Token` и отправить его на наш бэкенд. Это полностью устраняет необходимость отправлять администратора в веб-панель (Админку) для сложной OAuth-авторизации и подключения группы.
- Управление правами: запрос дополнительных прав доступа к сообщениям группы (для ботов/модерации).

**Пример вызова:**
```typescript
bridge.send('VKWebAppGetCommunityToken', {
  app_id: 54555042,
  group_id: 12345678, // ID группы (берем из launch_params: vk_group_id)
  scope: 'messages,manage,photos,docs'
})
.then((data) => {
  if (data.access_token) {
    console.log('Токен сообщества получен:', data.access_token);
    // Отправляем токен на бэкенд (POST /api/app/groups/me/token)
    // Бэкенд сохранит его в БД (groups.access_token) и сможет постить от имени этой группы!
  }
})
.catch((error) => {
  console.log('Ошибка получения токена сообщества:', error);
});
```

## 3. Проверка запрошенных прав (`VKWebAppCheckAllowedScopes` & `VKWebAppGetGrantedPermissions`)

Прежде чем пытаться выполнить действие, требующее определенного Scope, мы можем проверить, какие права пользователь уже нам предоставил.

**Сценарии использования:**
- Не показывать кнопку "Опубликовать", если пользователь еще не дал права `wall`.
- Мягко запросить недостающие права (`VKWebAppGetAuthToken`), если `VKWebAppCheckAllowedScopes` показывает, что их нет. Это избавляет от ошибок 15 (Access Denied) при попытках API-вызовов.

**`VKWebAppCheckAllowedScopes` (проверка конкретных скоупов):**
```typescript
bridge.send('VKWebAppCheckAllowedScopes', {
  scopes: 'wall,photos,groups'
})
.then((data) => {
  if (data.result) {
    data.scopes.forEach((scope) => {
      console.log(`Право ${scope.name} предоставлено: ${scope.allowed}`);
    });
  }
});
```

**`VKWebAppGetGrantedPermissions` (получение списка всех выданных прав):**
```typescript
// Работает для токенов пользователя и сообщества (добавляем ex. type="group", group_id=...)
bridge.send('VKWebAppGetGrantedPermissions')
.then((data) => {
  console.log('Выданные права:', data.permissions);
});
```

## Архитектурные преимущества для ZooPlatforma

1. **Бесшовный Onboarding Администраторов**: Сейчас администратору нужно заходить в `vk.zooplatforma.ru`, логиниться через ВК и подключать свою группу. С помощью `VKWebAppGetCommunityToken` мы можем сделать "Экран настройки" прямо внутри Mini App. Если администратор заходит в Mini App, мы видим его статус (он админ), предлагаем нажать "Подключить приложение", вызываем `VKWebAppGetCommunityToken` и отправляем токен на наш бэкенд. Готово! Группа подключена без переходов на сторонние сайты.
2. **Публикация от имени пользователя**: В будущем, если мы захотим позволить обычным пользователям делать репосты своих объявлений на личную стену (для большего охвата), мы сможем легко запросить `VKWebAppGetAuthToken` с правом `wall` и сделать `wall.post` от их имени.
3. **Безопасность UI**: Используя `VKWebAppCheckAllowedScopes`, мы можем динамически менять интерфейс: "Подключите сообщения, чтобы получать уведомления о модерации" -> если права нет, показываем красную плашку, если есть — зеленую.
